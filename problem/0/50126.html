50126. Line Up Numbers

I'm a slow walker, but I never walk backwards.

  • Status
  • Statistics

Task Description

There are two integer arrays AA and BB of sizes NN and MM respectively. The elements of AA and BB are in increasing order. Both A and B have a pointer array a and b of the same size. The pointer a[i]a[i] is pointing to A[i+1]A[i+1] and so on, and a[N−1]a[N−1] is NULL. That is, a[i]a[i] points to the next value in the AA after A[i]A[i]. The values of b[i]b[i] are similarly defined as in aa. Please refer to the following figure for an illustration.

sample

Now write a function to merge list A and B. That is, after the merging a[i] should point to the next elelement after A[i] in the list of the elements from both A and B. Similarly, b[i] should point to the next elelement after B[i] in the list of the elements from both A and B. After the merge we will have a pointer head pointering AA, since A[0]A[0] is 22 and is smaller than B[0]B[0]. Then a[0]a[0] should point to B[0]B[0], since B[0]B[0] is smaller than A[1]A[1]. Then b[0]b[0] should point to A[1]A[1], since it has the third smallest number 1010. a[1]a[1] should point to A[2]A[2], which is 1313, the fourth smallest number, and so on. Finally b[3]b[3] will have NULL since it is the largest number in both AA and BB.

Input Format

The first line has two integer NN and MM. They are sizes of array AA and BB. The second line has NN integers, which are elements of array AA. The third line has MM integers, which are elements of array BB. All of the numbers of AA and BB are unique.

Output Format

If your function is correct, the sample output will have only one line and that is “Finish!”. If your function is wrong, the sample output will show error messages. (If there are too many error messages, you will get "OLE".)

Subtasks

  • 10 points: Both arrays AA and BB have only one element.
  • 30 points: Either AA or BB has only one element.
  • 60 points: General cases.

Hint

For general cases you can use two integers to keep track of which part of AA and BB you have processed. If the one integer pointed by AA is smaller, then it is the next one in the list. After linking it to the end of the current list you increase AA and go on. If the one pointered by BB is smaller than you do the same for bb. You stop until one list stops. See the following pseudo code.

FindSmall ( int A[], int B[] )
{
    i = 0   // index of array A
    j = 0   // index of array B
 
    while ( i or j < size of A or B ):
        if  A[i] < B[j]:
            A[i] is the smallest number of remaining numbers.
            Connect the pointer you need to do.
            i++
        else
            B[j] is the smallest number of remaining numbers.
            Connect the pointer you need to do.
            j++
 
    if  i == size of A:
        for j to size of B:
            complete remaining numbers in B.
    else if  j == size of B:
        for i to size of A:
            complete remaining numbers in A.
}
