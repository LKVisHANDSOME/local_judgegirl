50158. Stop the Sequence

I'm a slow walker, but I never walk backwards.

  • Statistics

Task Description

Given a sequence of non-negative integers xx and a function C(x)C(x), find out the first three consecutive xx's whose C(x)C(x) are also consecutive integers.

The function CC is defined as follows. We will be given another function f(x)=(ax+b)modcf(x)=(ax+b)modc, and an interval [d,e][d,e]. For a given input xx, we will repeatedly apply ff on it. That is, we will consider the sequence xx, f(x)f(x), f(f(x))f(f(x)), f(f(f(x)))f(f(f(x))), ... and so on. We will stop when this process produces a value yy between dd and ee (inclusively). That is, d≤y≤ed≤y≤e. C(x)C(x) is deifned as the number of times we need to apply ff until it stops, and we guarantee that C(x)C(x) is a finite integer for our input xx. For example, if xx is between dd and ee then C(x)C(x) is 0. Also if f(f(f(x)))f(f(f(x))) is between dd and ee, but f(f(x))f(f(x)), f(x)f(x), and xx are not, then C(x)C(x) is 3.

You will be given a sequence of xx's and you need to compute their C(x)C(x). You will stop when the last three xx's give three consecutive C(x)C(x). Note that three consecutive numbers can appear in any order, like 7, 6, 5.

Input Format

The first three lines contain aa, bb, and cc. The next two lines contain dd and ee. The sixth line contains xx's. You must process all input until EOF.

Output Format

If you find the three consecutive CC's, output the three xx's in the order they appear. If you cannot find such numbers, output “Not found”.

Subtasks

  • 20 points: The three consecutive C(x)C(x) will appear as CC, C+1C+1, and C+2C+2.
  • 80 points: The three consecutive C(x)C(x) may appear in any order.
