50095. Lines of Words

I'm a slow walker, but I never walk backwards.

  • Status
  • Statistics
  • Solution

Task Description

Given NN lines of words, switch words among them. We will use (l,w)(l,w) to indicate the wthwth word in the lthlth line. Both line and word numbers are from 00. For the example (0,0)(0,0) is apple, (1,0)(1,0) is bird, and (2,2)(2,2) is frog for the following input.

apple
bird cat
dog elephant frog

Now we will process requests for switching words. A request (l1,w1,l2,w2)(l1,w1,l2,w2) indicates that we need to switch words (l1,w1)(l1,w1) and (l2,w2)(l2,w2). Note that we guarantee that the indices are all valid.

For example, if the request is (0,0,2,2)(0,0,2,2), then we switch apple with frog, and the lines change as follow. After finishing processing requests, output the final result in a line by line manner.

frog
bird cat
dog elephant apple

Subtask

NN is the number of lines, LiLi is the number of words in the ithith line, LL is the total number of words, ww is the max length of a word and WW is the total number of characters (including '\0') in all words.

  • 30 points: The input data is small.
    • N<32N<32
    • Li<32Li<32
    • L<1024L<1024
    • w<32w<32
    • W<50000W<50000
  • 40 points: WW is significantly large, so using "strcpy" function will cause TLE. Instead you should use a 2D pointer array, and switch the pointers only.
    • N<32N<32
    • Li<32Li<32
    • L<1024L<1024
    • w<1024w<1024
    • W<500000W<500000
  • 30 points: NN and the the maximum number of words in a line are extremely large, so allocating a 2D pointer array will cause MLE. You should use “two-level pointer”, as in the task last week, to implement the line and word structure. Also the number of characters in a word could vary dramatically, so you need to put all of them in a one-dimensional array to save storage. That is, whenever you read a word you advance the pointer to the next available position in that array. This can also save storage.
    • N<1024N<1024
    • Li<1024Li<1024
    • L<16384L<16384
    • w<1024w<1024
    • W<10000000W<10000000

Input Format

The input contains only one test case. The first line contains one integer NN, representing the number of lines. The next NN lines contains LiLi words. After N+1N+1 lines, there is a line contains one integer MM, representing the number of requests. The next MM lines contains four integers l1l1, w1w1, l2l2, w2w2.

Output Format

Print the NN lines in the table in each line, and each word in one line is separated by a single space. And a 'newline' after the last word in each line.

Sample Input 1

3
apple
bird cat
dog elephant frog
1
0 0 1 0

Sample Output 1

bird
apple cat
dog elephant frog
