50190. Set Sorting and Searching

I'm a slow walker, but I never walk backwards.

  • Status
  • Statistics
  • Solution

Task Description

Remember that in the Hitting Set problem last week we need to sort the sets to hit according to their largest element. This task will demonstrate how to sort these sets. That is, there are NN different sets of positive integers, and you need to sort these NN sets by their largest elements in ascending order. If the largest elements of two sets are the same, then we compare the second largest elements, and so on. If all the largest elements of one set are the same as the largest elements of another set with more elements, the set with more elements is larger.

After sorting these NN sets, you will have to find out whether a set appears in the sorted NN sets. You should output the index of the set if it is in the sets, otherwise you should output −1−1. The sorted NN sets are indexed from 00 to N−1N−1.

Note

You will get TLE if you use time consuming sorting methods such as bubble sort. You will also get TLE if you use linear search the set to find a match. Also you should dynamically allocate memory for every set since NN is very large and the number of elements of some sets could be very large.

Input Format

The first line has an integer NN, which is the number of sets.

The following NN lines represent NN sets. In each line, the first number ss is the size of the set, and the following ss numbers are the elements of the set.

Next, there is a number MM, and the following MM lines are the sets you need to find. The format of the sets are the same as above.

  • 0<N≤100000<N≤10000
  • 0<s≤10000<s≤1000
  • 0<M≤200000<M≤20000

Output Format

You should output the search results in MM lines.

Subtasks

  • 20 points: you can use bubble sort and linear search.
  • 40 points: you can use quick sort and linear search.
  • 40 points: general cases.
