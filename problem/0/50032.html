50032. N-pieces (special judge)

I'm a slow walker, but I never walk backwards.

Problem Description

Given a nn by nn chessboard, is it possible to place mm pieces in it so that no more than kk pieces appear in a row, in a column, or along a 45 degree line? Note that there are two 45 degree lines for every cell (x,y)(x,y) not in the corners -- the first one goes through (x−1,y+1)(x−1,y+1), (x,y)(x,y), (x+1,y−1)(x+1,y−1), and the second one goes through (x+1,y+1)(x+1,y+1), (x,y)(x,y), (x−1,y−1)(x−1,y−1).

Subtasks

  • 5pt. n=3n=3, m=4m=4, and k=2k=2.
  • 5pt. n=3n=3, m=3m=3, and k=1k=1.
  • 15pt. n=6n=6, m=6m=6, and k=1k=1.
  • 20pt. nn is no more than 8, mm is no more than 16, and kk is no more than 1.
  • 50pt. nn is no more than 12, mm is no more than 30, and kk is no more than 10.
  • 5pt. nn is no more than 15, mm is no more than 100, and kk is no more than 100.

Hints

  • You try to place the pieces into each cell in a row-by-row manner. Each time you place a piece, you need to check if your placement is valid.
  • There are various optimizations you can use. For example, if you have placed kk pieces in the same row, you should go on to the next row.
  • To get the last 5 points, you need a careful implementation that tracks the number of pieces already placed in each row, each column, and each 45 degree line.

Input Format

Each line has nn, mm, and kk for a test case. You need to process all test cases until EOF.

Output Format

If it is possible to place mm pieces in the chessboard, print one possible solution where o is a piece and . is an empty square. If it is impossible to place mm pieces in the chessboard, print N.
