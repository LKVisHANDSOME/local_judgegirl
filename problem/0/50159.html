50159. Stock Price

I'm a slow walker, but I never walk backwards.

Task Description

Given a sequence of stock prices, you need to find out the longest length of given patterns. For simplicity we assume that two consecutive prices are different. There are two patterns to look for -- "valley" and "mountain". A sequence (a1,...,ai,...,an)(a1,...,ai,...,an) is a valley if (a1,...,ai)(a1,...,ai) is decreasing and (ai,...,an)(ai,...,an) is increasing. A sequence (a1,...,ai,...,an)(a1,...,ai,...,an) is a mountain if (a1,...,ai)(a1,...,ai) is increasing and (ai,...,an)(ai,...,an) is decreasing.

Your program will output the starting point and the length of the longest mountain or valley. If there are more than one sequence that have the longest length, choose the one with the longest left sequence. If the lengths of the left sequences are still the same, output the one that appears first.

Here is an example. If the sequence is [6,5,4,5,4,3][6,5,4,5,4,3], it contains a valley [6,5,4,5][6,5,4,5] and a mountain [4,5,4,3][4,5,4,3]. Both are 4 in length. However, you need to choose [6,5,4,5][6,5,4,5] because it has a longer left sequence length 33.

If you cannot find any valley or mountain (the whole sequence is increasing or decreasing, or it contains nothing), you should output 00.

Input Format

The input is x1,x2,…,xnx1,x2,…,xn, a sequence of integers, and you should process all of them until EOF. It is guaranteed that any two consecutive numbers are different.

  • 0≤xi<20000, i=1,2,…,n0≤xi<20000, i=1,2,…,n
  • 0≤n<400000≤n<40000

Output Format

Output the length of the longest valley or mountain and the index of the first element of that pattern. The index starts from 11. If there are no valleys or mountains, output 00.

Sample Input 1

6 5 4 5 4 3

Sample Output 1

4 1

Sample Input 2

1 2 3 4 5 6 7 8 9

Sample Output 2

0
