50003. Turtle Graphics

I'm a slow walker, but I never walk backwards.

  • Status
  • Statistics
  • Solution

Problem description

There is an old drawing program called turtle graphics. The program is given a series of LL lines and it will draw them on an XX by YY matrix. A line consists of a series of nn points, and the program will draw a straight line between two consecutive points. For simplicity we assume that the each segment of a line is either vertical, horizontal, or 45∘45∘ diagonal.

The input is as follow. The first line has LL, XX, and YY. All of them are between 1 and 100. Each of the next LL lines starts with nn, the number of points in that line, and the next nn pairs of coordinates indicate the position of the points in the that line. Note that nn is positive and could be very large so you cannot store the coordinates in an array. If you are given a point that is not in the grids, or does not form a vertical, horizontal, or 45∘45∘ diagonal line with the previous point, report the line number and point number and stops the program.

If the input is correct, then the output has XX lines, each of them has YY numbers. You should output a 11 if that cell is drawn, 00 otherwise. If the input is incorrect report the line number and point number of the point that is causing the problem. Both line number and point number start from 11.

Technical Specification and constraints

  • 10 pt. LL is 11 and the line is a valid horizontal line, i.e, nn is 22.
  • 30 pt. LL is 11 and the lines are only horizontal and vertical, nn is between 11 and 1010, and the input is always correct.
  • 40 pt. LL is between 11 and 100100 and the lines could be vertical, horizontal, or 45∘45∘ diagonal, nn is between 11 and 1010, and the input is always correct.
  • 10 pt. LL is between 11 and 100100 and the lines could be vertical, horizontal, or 45∘45∘ diagonal, nn is between 11 and 1010, and input could be incorrect.
  • 10 pt. LL is between 11 and 100100 and the lines could be vertical, horizontal, or 45∘45∘ diagonal, nn could be very large so you CANNOT store the points in an array, and input could be incorrect.
