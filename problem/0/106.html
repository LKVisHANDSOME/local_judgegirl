106. Divisible

I'm a slow walker, but I never walk backwards.

Task Description

Write a program to determine if a number is divisible by 2,3,52,3,5, and 1111. The rules are as follow.

  • A number is divisible by 22 if the last digit is divisible by 22.
  • A number is divisible by 33 if the sum of the digits is divisible by 33.
  • A number is divisible by 55 if the last digit is 00 or 55.
  • A number is divisible by 1111 if the difference between the sum of the even positioned digits and the the sum of the odd positioned digits is divisible by 1111.

For example the number 190949190949 is not divisible by 22 because 99 is not divisible by 22. It is not divisible by 33 because 1+9+0+9+4+9=321+9+0+9+4+9=32 is not divisible by 33. It is not divisible by 55 because the last digit is 99. It is divisible by 1111 because the sum of even positioned digits is 9+9+9=279+9+9=27, and the the sum of odd positioned digits is 1+0+4=51+0+4=5. The difference between 2727 and 55 is 2222, which is divisible by 1111.

Limits

The number of digits in a number is no more than 10001000. Note that the number of digits could be very large so you cannot store the number in an int.

Input

The input has several lines. Each line has a positive integer. A -1 indicates the end of input.

Output

For each input number your program should output four yes or no, which are separated by a space character. These yes and no indicate whether the input number is divisible by 2,3,52,3,5, and 1111.

Sample input

190949
20
-1

Sample output

no no no yes
yes no yes no
