50121. Push Stones

I'm a slow walker, but I never walk backwards.

  • Status
  • Statistics

This is a game about a stone pushing.

Task Description

We have an NN by MM map with one player and WW stones. The vertical index is from 00 to N−1N−1, and the horizontal index is from 00 to M−1M−1.

The player will act according to the instructions. Each instruction is a number from 00 to 44. Let the current position of the player be (x,y)(x,y). The following is the action the play should perform when he receives that instruction.

  • If the instruction is 00, output the current map information.
  • If the instruction is 11, the person walks from (x,y)(x,y) to (x+1,y)(x+1,y).
  • If the instruction is 22, the person walks from (x,y)(x,y) to (x,y+1)(x,y+1).
  • If the instruction is 33, the person walks from (x,y)(x,y) to (x−1,y)(x−1,y).
  • If the instruction is 44, the person walks from (x,y)(x,y) to (x,y−1)(x,y−1).

The coordinate system is defined as follows:

Coordinate

There are also stones in the map. Every stone is in its own cell and has a weight. A player also has an amount of energy. When the player moves into an empty cell, his energy will not change. However, if the player moves and encounters a stone, he needs to push it, along with all the stones behind it, all the way to the first empty cell in that direction. If the player’s energy is at least the sum of the weights of the stones he has to push, then he can push them. That is, he and the stones he pushed will move one cell in that direction. After he pushes the stones, his energy will decrease by the sum of the weights of the stone he pushed. On the other hand, if the player’s energy is less than the sum of the weights of the stone he wants to push, then he cannot push them, and he will stay in same cell. Also note that the player will ignore any instruction that causes the person or any stone to be out of the map.

Write a program to simulate this game, and output the map information when given instruction 00.

Subtask

  • 10 points: There are no stones in the map.
  • 20 points: There is only one stone in the map.
  • 70 points: There could be multiple stones in the map.

Input Format

The input contains only one test case. The first line contains two integers NN and MM, which are the vertical and horizontal size of the map. The second line contains three integers, the player’s row index, column index, and energy. The third line contains one integer OO representing the number of stones. Each of the following OO lines contains three integers for row index, column index, and the weight of stones. The last line contains the instructions. You need to process them until the end of file. The number of instructions is at least one.

  • 0<N,M≤5000<N,M≤500
  • 0≤O<N×M0≤O<N×M

Output Format

Print the current map information if the instruction is 00. This information includes the person’s energy and the weight of stones. The rest of cells are zero.

Note that there is no space at the end of line.
