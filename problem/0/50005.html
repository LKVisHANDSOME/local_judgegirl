50005. Pattern Recognition

I'm a slow walker, but I never walk backwards.

  • Status
  • Statistics
  • Solution

Problem Description

Write a program to recognize a kk by kk pattern in an nn by nn matrix. Both the matrix and the pattern consist of integers. A submatrix of the matrix matches the pattern if the following are true.

  • There are no more than mm mismatch among the numbers.
  • The sums of the numbers in the two matrices differ no more than dd.

For example the following pattern is a match of the following submatrix if mm is 22, and dd is 22. However, they do not match when either mm is 11, or dd is 11.

1 2 3
4 5 6
7 8 9
1 2 3
4 7 6
7 8 5

Now given the matrix, the pattern, kk, and dd, please report the position of upper-left corner of the last match. Here we assume that the first row is at the top and the first column is on the left. The last match is the match that has largest row index with its upper left corner element. If there are more than one match in that row, report the one with the largest column index. All indices start from 0.

In some subtasks there could be no match. In that case report -1 -1.

Subtasks

  • 20pt. nn is between 11 and 1010, kk is 11, mm is 00, dd is 00, and there is always a match. That is, you only need to check where the last pattern is.
  • 20pt. nn is between 11 and 1010, kk is between 11 and nn, mm is between 11 and k×kk×k, dd is extremely large so you do not to care about it, and there is always a match.
  • 30pt. nn is between 11 and 1010, kk is between 11 and nn, mm is between 11 and k×kk×k, dd is positive, and there is always a match.
  • 30pt. nn is between 11 and 1010, kk is between 11 and nn, mm is between 11 and k×kk×k, dd is positive, and there could be no match.

Input Format

The first line contains four integers nn, kk, mm, dd. The next nn lines has the matrix and nn elements in each line. The next kk lines has the pattern and kk elements in each line.
